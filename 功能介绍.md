# Emon DSL 功能介绍

## 项目概述

**Emon DSL** 是一套面向 eBPF 监控场景的领域特定语言及其编译工具链。通过高层声明式语法，将常见的 eBPF 监控需求抽象为可复用的语言原语，使开发者无需直接编写底层 C/eBPF 代码即可完成可观测性工具的开发。

---

## 核心功能模块

### 1. 词法分析器 (Lexer)

**功能定位**：将源代码文本转换为 Token 序列

**支持的 Token 类型**：

| 类别 | 示例 | 说明 |
|------|------|------|
| **关键字** | `tool`, `observe`, `where`, `measure`, `when`, `emit`, `every` | 语言保留字 |
| **标识符** | `target_pid`, `min_latency` | 用户定义的变量名 |
| **聚合标识符** | `@count`, `@avg_latency` | 以 `@` 开头的聚合变量 |
| **常量** | `42`, `100us`, `256KB`, `"hello"`, `true` | 整数、时间、大小、字符串、布尔值 |
| **运算符** | `+`, `-`, `*`, `/`, `==`, `!=`, `&&`, `\|\|` | 算术、比较、逻辑运算符 |
| **分隔符** | `{`, `}`, `(`, `)`, `[`, `]`, `;`, `,` | 语法结构符号 |

**错误处理**：
- 非法字符识别
- 未闭合字符串检测
- 未闭合注释检测
- 位置信息准确报告

### 2. 语法分析器 (Parser)

**功能定位**：将 Token 序列构建为抽象语法树 (AST)

**支持的语法结构**：

```ebpf
tool syscall_latency_monitor {
    option target_pid = 0;
    option min_latency = 100us;
}

observe syscall("read", "write", "openat")
where pid > 0
measure latency
when latency > min_latency
{
    @count[comm, pid] = count();
    emit { time = nsecs; pid = pid; };
}

every 1s {
    print(@count);
}
```

**支持的观测目标**（7种）：
- `syscall` - 系统调用追踪
- `kernel` - 内核函数追踪
- `tracepoint` - 内核追踪点
- `uprobe` - 用户态函数追踪
- `sched` - 调度器事件
- `file` - 文件系统事件
- `net` - 网络事件

### 3. 语义分析器 (Semantic)

**功能定位**：验证 AST 的语义正确性

**检查项**：
- ✅ 变量作用域验证（hook 上下文变量在正确位置使用）
- ✅ 聚合函数参数检查（`avg()`, `sum()` 需要参数，`count()` 不需要）
- ✅ 重复定义检测（同一作用域内不允许重复定义变量/聚合）
- ✅ 测量指标作用域（`latency`, `retval`, `size` 必须在 `measure` 子句中声明）
- ✅ 表达式阶段验证（`retval` 不能在 `where` 子句中使用）

### 4. 中间表示 (IR)

**功能定位**：将 AST 转换为与目标无关的中间表示

**IR 组件**：
- **Probe Generation** - 探测点生成
- **Map Generation** - BPF Map 生成与去重
- **Emit Generation** - 事件输出定义
- **Lifecycle Management** - begin/end/every 生命周期管理
- **JSON Serialization** - IR 的 JSON 序列化支持

## 使用方法

### CLI 工具

```bash
# 词法分析模式
python3 main.py examples/syscall_latency.emon

# 语法分析模式（生成 AST）
python3 main.py --parse examples/syscall_latency.emon

# 交互式 REPL
python3 main.py
```

### 示例程序

```ebpf
# syscall_latency.emon - 系统调用延迟监控
tool syscall_latency_monitor {
    option target_pid = 0;
    option min_latency = 100us;
    option interval = 1s;
}

observe syscall("read", "write", "openat")
where target_pid == 0 || pid == target_pid
measure latency
when latency > min_latency
{
    @count[comm, pid, syscall] = count();
    @avg_latency[comm, pid, syscall] = avg(latency);
    
    emit {
        time = nsecs;
        comm = comm;
        pid = pid;
        latency = latency;
    };
}

every interval {
    print("==== summary ====");
    print(@count);
    print(@avg_latency);
}
```

## 技术特点

1. **声明式语法**：无需编写底层 eBPF C 代码
2. **多目标支持**：支持 7 种 eBPF 观测目标
3. **完整工具链**：从词法分析到代码生成的完整编译流程
4. **类型安全**：完善的语义检查确保程序正确性
5. **可扩展性**：模块化设计便于添加新特性
