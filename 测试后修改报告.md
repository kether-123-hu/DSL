## 三、问题修复

### 3.1 语法文件修复 (grammar/emon.lark)

**问题**：`measure` 子句中的多个指标无法正确解析

**修改位置**：第46-50行、第134-138行

**修改前：**
```lark
// Measure keywords (higher priority than IDENT)
MEASURE_KW.2: "latency" | "count" | "size" | "retval" | "stack"

// Regular identifier: letter or underscore start, may contain digits.
IDENT: /[a-zA-Z_][a-zA-Z0-9_]*/

// MeasureItem ::= "latency" | "count" | "size" | "retval" | "stack"
measure_item: MEASURE_KW
```

**修改后：**
```lark
// Measure keywords
MEASURE_KW: "latency" | "count" | "size" | "retval" | "stack"

// Regular identifier: letter or underscore start, may contain digits.
IDENT: /[a-zA-Z_][a-zA-Z0-9_]*/

// MeasureItem ::= "latency" | "count" | "size" | "retval" | "stack"
measure_item: "latency" | "count" | "size" | "retval" | "stack"
```

**问题原因**：
- 优先级 `.2` 定义导致 `count` 等词与 IDENT 冲突
- `count()` 聚合函数无法被正确识别

**解决方案**：
- 移除优先级定义
- 直接在 `measure_item` 规则中使用字符串字面量

### 3.2 解析器修复 (emon/parser.py)

**问题**：measure_clause 和 measure_item 处理逻辑不正确

**修改位置**：第196-209行

**修改前：**
```python
def measure_clause(self, items):
    metrics = [item for item in items[1:] if isinstance(item, Metric)]
    return MeasureClause(metrics=metrics)

def measure_item(self, items):
    return _METRIC_MAP[items[0].value]
```

**修改后：**
```python
def measure_clause(self, items):
    metrics = []
    for item in items[1:]:
        if isinstance(item, Metric):
            metrics.append(item)
    return MeasureClause(metrics=metrics)

def measure_item(self, items):
    for item in items:
        if isinstance(item, Token):
            return _METRIC_MAP[item.value]
        elif isinstance(item, str):
            return _METRIC_MAP[item]
    return _METRIC_MAP[str(items[0])]
```

**改进说明**：
- 增加对字符串类型和 Token 类型的兼容性处理
- 修复多指标解析时的过滤逻辑
